What you are building

Local DCA normally skips login via dev shortcuts. This setup turns those off and wires the full flow:

StepWhat happens
1Visit DCA → redirect to Unified Login
2Username / password
3WCF services on devsvcs.newportgroup.com
4OTP / MFA (may use /userprofile)
5SAML SSO → YARP → PDXWeb → Dashboard

Architecture

dev.newportgroup.com  (127.0.0.1)   ← browser entry
├── /              → DCA.Yarp
├── /login         → UnifiedLogin.FrontEnd.Mvc
└── /userprofile   → EUM UserProfile.Mvc

dev.pdxweb.com  (127.0.0.2)         ← YARP backend
├── /              → PDXWeb
└── /login, /app, /legacy, …

devsvcs.newportgroup.com  (127.0.0.1)
├── NGUnifiedLogin.WcfService
├── EnterpriseUserManagement.WcfService + .Api
└── SSOIntegration.WcfService

Setup steps (do these first)

Step 1 — Repos and Cursor

  • Clone as sibling folders: DCA, UnifiedLogin, EnterpriseUserManagement, NewportDeveloperSetup
  • Open Cursor on the parent folder (not inside one repo)
  • Use Agent mode · Windows 10/11 + IIS

Step 2 — One-time machine setup (Administrator)

Run once if you never configured this machine:

cd <your-root>\NewportDeveloperSetup\Scripts
.\DeveloperMachineSetup.ps1

Run at least: IIS config, Connection Strings, Public apps, Services, Host file entries, SSO certificate.

If dev.pdxweb.com apps return 503 (old IIS layout):

cd <your-root>\DCA\.localSetup\Scripts
.\Migrate-IIS-PdxwebYarp.ps1

Step 3 — Hosts file

EntryRequired
127.0.0.1 → dev.newportgroup.com, devsvcs.newportgroup.comYes
127.0.0.2 → dev.pdxweb.comYes
127.0.0.1 → devsqlNo — breaks SQL

Step 4 — VPN, SQL, and app pool password

  • Connect VPN if your team requires it (Redis + SQL)
  • Test SQL: sqlcmd -S devsql -d EnterpriseUserManagement -E -Q "SELECT 1" → must return 1
  • Run as Admin (very common 503 fix): DCA\.localSetup\Scripts\ResetAppPoolCredentials.ps1

Step 5 — Disable auto-logon and set local login URLs

Edit DCA\PDX\PDXWeb\DeveloperAppSettings.json:

  • _AutoLogonPersonId — empty (not AutoLogonPersonId)
  • external-login-participant-url → http://dev.newportgroup.com/login/Participant

Step 6 — Build projects

  • npm install + npm run build in UnifiedLogin\Source\UnifiedLogin.FrontEnd.Mvc
  • dotnet build: DCA.Yarp, PDXWeb, EUM Api + WcfService, UL WcfService, SSOIntegration WcfService

Step 7 — YARP and UnifiedLogin config

  • DCA\MFE\DCA.Yarp\web.config → hostingModel="OutOfProcess"
  • UnifiedLogin DeveloperAppSettings.json — keys must be inside "AppSettings": { } (not root level); include Redis connection string if sessions fail

Step 8 — SSO certificate and EUM database

  • Install dev-local cert or run SetupLocalDevelopmentCertificate.ps1 from NewportDeveloperSetup
  • On shared devsql, EUM rule 4: cert 177, IssuerId https://dev.newportgroup.com/login

Step 9 — Permissions and recycle IIS

  • If 500.19: Admin icacls on Yarp + UnifiedLogin folders for IIS_IUSRS
  • iisreset /restart and start app pools (DCA.Yarp, DCA, UnifiedLogin, devsvcs WCF pools)
